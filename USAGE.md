# Jarvis Virtual-COS — Руководство пользователя и разработчика

---

## Введение
Jarvis Virtual-COS — интеллектуальный советский ассистент с глубокой когнитивной архитектурой.  
Проект реализует многоуровневую память, адаптивное самообучение и интерактивный веб-интерфейс для быстрого обучения.

---

## Основные компоненты

- **main.py**  
  Запуск веб-панели, TCP-сервера, когнитивного цикла и интеграция модулей.

- **memory/**  
  Хранение и обработка смысловых ассоциаций.

- **interface/**  
  Обработка пользовательских команд и интерактивных вводов.

- **meta/**  
  Meta-слои смыслового поля — `field7d.py`, `field11d.py`, `field15d.py`.

---

## Установка и запуск

1. Клонируйте репозиторий:
git clone https://github.com/KHUSHVAKHT175/Jarvis-Virtual-COS.git
cd Jarvis-Virtual-COS

2. Установка зависимостей (если есть `requirements.txt`):
pip install -r requirements.txt

3. Запуск Jarvis:
python main.py

4. Откройте браузер на [http://localhost:8080/](http://localhost:8080/) для работы с веб-интерфейсом.

---

## Использование

- Вводите команды вручную в веб-форме и нажимайте "Отправить".
- Для быстрого пакетного обучения нажмите кнопку "Обучить Папуаса".
- Отслеживайте параметры системы — пульс, энергию смыслов, цели и логи.

---

## Как расширить и обучать

- Указывайте новые команды в массиве `batch_commands` в `main.py` для пакетного обучения.
- Используйте методы из `memory.py` для работы с памятью.
- Добавляйте новые мета-слои и когнитивные функции для расширения возможностей.

## FAQ

**В: Как добавить свою команду для обучения?**  
О: Добавьте в `batch_commands` в `main.py`.

**В: Можно ли использовать Jarvis с облачными сервисами?**  
О: В текущем виде Jarvis работает локально, но можно интегрировать облачные API дополнительно.

**В: Как проверить состояние памяти и энергию смыслов?**  
О: Информация отображается в веб-интерфейсе на главной странице.
